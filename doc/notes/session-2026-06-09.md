## 2. 传感器校准页 — OD0/OD10/OD50 添加 Calibrate 按钮

### 背景
SP90 产品传感器校准页有 4 个 Outer Diameter 字段：
- **Outer Diameter AD** — 实时传感器读数，只读显示
- **Outer Diameter AD0** — 校准存储值，可手动键盘输入
- **Outer Diameter AD10** — 同上
- **Outer Diameter AD50** — 同上

需求：在 AD0/AD10/AD50 右下角各加一个"校准"按钮，点击后将当前 AD 实时值复制到对应字段。

### 实现方案
文件：`surface/src/ui/sensor_calib.c`

只改 surface，不动 scence（后端）。利用现有 `RENA_UI_CONFIG_SET_REQ` 消息和 `RENA_CONFIG_INDEX_CALIB_ODx` 配置索引。

#### 回调逻辑
```c
static void rena_major_btn_od0_event_cb(lv_event_t *e)
{
    if (lv_event_get_code(e) == LV_EVENT_CLICKED)
    {
        rena_ui_config_set_req req;
        req.index = RENA_CONFIG_INDEX_CALIB_OD0;
        req.type = RENA_CONFIG_TYPE_UINT32;
        req.is_valid = true;
        req.accept_soft_limit = true;
        req.config_value.uint32_value = rena_get_outer_diameter();
        rena_send_msg(RENA_UI_CONFIG_SET_REQ, &req, sizeof(req));
    }
}
```

#### 改动点
| # | 位置 | 内容 |
|---|------|------|
| 1 | forward declarations | 加 3 个 `btn_od0/1/2` 函数声明 |
| 2 | `area_od0` 后 | 创建 `btn_od0` + label "STR_CALIBRATE" |
| 3 | `area_od1` 后 | 创建 `btn_od1` |
| 4 | `area_od2` 后 | 创建 `btn_od2` |
| 5 | 事件注册区 | 注册 3 个 btn callback |
| 6 | 文件末尾 | 加 3 个 callback 实现 |

#### Struct 说明
OD 相关 area 不需要存入 `ui` struct。现有的 ml area 存储（`ui.area_1ml_100kp` 等）在整个工程中仅赋值、从未读取使用。

## 3. OD 值数据流分析（scence → surface）

### 完整链路
```
DB(JSON) → calib_param → send_nty() → SystemV msg queue → proc_notify → global.c struct
                           250ms周期                                        ↑
                           timer_cb  ←────────────────────────────────────────┘
                                    ↓
                             lv_label_set_text
```

### 逐层说明

#### scence 端
1. **启动加载**：`calib_setting.cpp:rena_set_calib_setting()` 从 JSON 读取 `od0/od1/od2` → 存入 `rena_calib_param`
2. **定时推送**：`timer_task_thread.cpp` 每 250ms 调用 `send_nty()`
3. **组装数据**：`ui_sensor_data.cpp:send_nty()` 从 `calib_param->get_outer_diameter0()` 读值，填入 `rena_ui_sensor_data_nty` 结构体
4. **发送消息**：`rena_send_msg_to_ui(RENA_UI_SENSOR_DATA_NTY, ...)`

#### surface 端
1. **接收消息**：`msg_proc.c` 分发 → `proc_notify.c:rena_proc_ui_msg_sensor_data_nty()`
2. **解包赋值**：`rena_set_outer_diameter0(sensor_data_nty->outer_diameter0)`
3. **内存存储**：`global.c` 中 `rena_sensor_data_params.outer_diameter0 = od`
4. **UI 定时更新**：`sensor_calib.c` 中 250ms 定时器 `rena_sensor_calib_info_update_time_cb` → `lv_label_set_text_fmt(ui.label_od0, "%d", rena_get_outer_diameter0())`

### 用户设置新值时的写入路径
```
surface → RENA_UI_CONFIG_SET_REQ(index=ODx, value=xxx)
→ scence: ui_msg_proc_config_set.cpp
  → calib_param->set_outer_diameter0(int16_t)  // 更新内存
  → rena_set_calib_setting()                    // 写回DB持久化
```

### 关键区别
| 字段 | 来源 | 性质 | 持久化 |
|------|------|------|--------|
| `outer_diameter` (AD) | `sensor_param.get_outer_diameter()` | 实时传感器读数 | 否 |
| `outer_diameter0/1/2` (ODx) | `calib_param.get_outer_diameter0/1/2()` | 校准存储值 | 是 (JSON/DB) |

## 4. 报警栏超长滚动 Bug 排查与修复

### 现象
修改 `alarm_poll.c` 实现报警文本超长水平滚动后，烧录到泵上运行时出现：电机启动（绿灯亮）但 UI 不切换到运行页面，无运行动画。

### 排查过程

#### 代码调用链
```
user Start → ready_rate_start_event_cb
  → RENA_UI_RATE_READY_START_REQ → scence 启动电机
    → scence 回复 RENA_UI_RATE_READY_START_RSP
      → proc_rate.c:rena_proc_ui_msg_rate_ready_start_rsp
        → switch_main_page(ui_flag.main_page_id)  // 页面切换
          → run_rate_load(main_page)
            → rena_alarm_poll_create(page)  // 二次加载 alarm_poll
```

#### 根本原因
`rena_alarm_poll_create` 中 `memset(&ui, 0, sizeof(ui))` 在 `rena_alarm_poll_destory()` **之前**执行，导致：

1. `ui` 中所有指针（包括旧页面 LVGL 对象引用）被清零
2. `rena_alarm_poll_destory()` 被调用时：
   - `lv_anim_del(NULL, NULL)` — 遍历全局动画链表
   - `lv_style_reset(&alarm_list_style.*)` — 重置了**正在被旧页面使用的样式**
3. 旧页面正在进行消失动画（`_rena_pm_disappear`），其 LVGL 对象引用已被重置的样式
4. 旧页面重绘时访问已释放的样式内存 → **heap 损坏**
5. heap 损坏后新页面（运行页）无法正常加载

#### 次要问题
- `expand_anim`/`collapse_anim` 未声明为 `static`，有符号污染风险
- `destroy` 函数没有 NULL 指针保护

### 修复方案

#### 核心修复（3 点）
| # | 修改 | 位置 |
|---|------|------|
| 1 | `memset` 移到 `destroy` 之后 | `rena_alarm_poll_create` |
| 2 | `destroy` 中 `lv_anim_del` 加 NULL 判断 | `rena_alarm_poll_destory` |
| 3 | `expand_anim`/`collapse_anim` 加 `static` | 文件顶部 |

#### 可选优化
`ready_rate_unload` 中补充调用 `rena_alarm_poll_destory()`，确保旧页面的 alarm_poll 被彻底清理。

### 关于 Flex 布局
排查过程中曾怀疑 `lv_obj_set_flex_flow` 未使能，但确认 `top_bar.c` 已使用 Flex，目标板 `lv_conf.h` 中 `LV_USE_FLEX` 已开启。Flex 布局本身不是问题原因，可以保留。
